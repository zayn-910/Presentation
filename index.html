Core_Segments

System Architecture

Every HTML document is split into two critical zones. The brain and the body.

  • <HEAD> The Brain. Contains metadata, titles, links to CSS/JS. Invisible to users.
  • <BODY> The Physical Form. Contains all visible content (text, images, buttons).

The Skeleton

Raw Structure

HTML is the raw concrete. It carries the weight of the web. Without it, CSS is just paint on thin air.

Bones don't need to be pretty. They need to be strong.

Semantics_

// NON-SEMANTIC (BAD) <div class="nav">Menu</div> <div class="footer">End</div>
// SEMANTIC (GOOD) <nav>Menu</nav> <footer>End</footer>

Speak to the Machine

Semantic HTML conveys meaning to search engines and screen readers. It tells the machine WHAT the content is, not just where it sits.

> A <div> means nothing.

Attributes

System Controls

Attributes provide additional information about elements. They are the switches and dials of your HTML tags.

<img src="logo.png" alt="Brand" /> <a href="/login" class="btn"> <input type="password" required />

Element_Types

Block

Starts on a new line. Consumes full width. Examples: div, p, h1, section, footer.

Inline

Flows with text. Takes necessary width. Examples: span, a, strong, em.

Void

Self-closing. No content inside. Examples: img, br, input, hr, meta.

Input_Terminals

User Interaction

Forms are the primary method for collecting user data. Without them, the web is read-only.

  • <form>Wrapper for data

Initiating Animation Subsystems...

Why Animation?

Attention & Feedback

Animation is not just decoration. It guides the user's eye, provides feedback on interactions (clicks, hovers), and masks loading times.

  • Enhances User Experience
  • Reduces Cognitive Load
  • Creates Continuity

Keyframes

@keyframes glitch {   0% { transform: translate(0); }   20% { transform: translate(-2px, 2px); }   40% { transform: translate(-2px, -2px); }   100% { transform: translate(0); } }
.element {   animation: glitch 0.5s infinite; }

Defining the Sequence

@keyframes allow you to define the intermediate steps in a CSS animation sequence. You control the start (0%), the end (100%), and any waypoints in between.

Methods

Transitions

Simple change from State A to State B. Triggered by hover or focus. Efficient and easy.

Animations

Complex sequences. Can loop infinitely. Does not require a trigger (runs on load).

Transforms

Moving, scaling, rotating elements. Hardware accelerated for smooth 60fps performance.
